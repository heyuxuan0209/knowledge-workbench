# Handoff: Knowledge Workbench（知识工作台）

## Overview
面向「一人媒体公司」的 AI 内容流水线工作台。价值链：信息输入 → 理解 → 形成观点 → 生产内容 → 分发。**产品终点是「产出带观点的内容」**，所有功能向这个终点收敛。本原型是一个桌面 Web 单页应用（宽屏三栏），覆盖六阶段流水线：① 采集 → ② 理解 → ③ 沉淀 → ④ 洞察 → ⑤ 创作 → ⑥ 发布。

> 六阶段是**产品心智模型**，不是导航条目。导航按功能划分（见下）。

## About the Design Files
本包内的 `Knowledge Workbench.dc.html` 是**用 HTML 制作的设计参考**（高保真原型，展示预期外观与交互），**不是可直接上线的生产代码**。它是一个 "Design Component"（`.dc.html`），运行依赖一个私有 runtime（`support.js`），脱离该环境无法直接打开——请把它当作**视觉与交互的规格说明**来读，配合本 README 在目标代码库里用其既有技术栈（React / Vue / SwiftUI / 原生等）和既有组件/设计系统**重新实现**。若项目尚无前端环境，选择最合适的框架实现即可。不要把这份 HTML 直接搬进产品。

## Fidelity
**高保真（hifi）**。颜色、字体、间距、圆角、交互均为最终意图，请按像素级还原，但用你代码库里已有的组件库与模式来搭。

## 全局布局（三栏）
- 顶栏 `header`：高 56px，左侧品牌（11px 圆点 #3d5a80 + 衬线标题「知识工作台」18px/600），右侧头像圆点 30px（#3d5a80，白字）。底边 1px `rgba(33,31,26,.1)`。
- 左栏 `nav`：默认宽 196px，可折叠到 62px（仅图标）。顶部一个折叠箭头（chevron，折叠时旋转 180°）。分两组：
  - 上组：资讯 / 素材 / 主题 / 创作
  - 底部组（`margin-top:auto` + 上边框分隔）：信源 / 设置
  - 每项：16px 描边图标（`stroke=currentColor`）+ 13.5px 标签；当前项底色 `rgba(61,90,128,.12)`、文字 #3d5a80/600。折叠时标签 `display:none`。
- 中栏 `main`：`flex:1`，内容 `max-width:720px` 居中，`overflow-y:auto`。按当前导航切换页面。
- 右栏 `panel`：默认宽 322px（创作页为 334px），可折叠为 40px 细条（点箭头展开）。内容随页面变化（见「右侧面板」）。

## 导航与页面（中栏）
状态 `page ∈ {feed, notes, sources, topics, studio, settings}`（reports 由 feed 内入口进入，无独立导航项）。

### 1. 资讯 feed（Feed 主页）
- **万能收口栏**：输入框「粘贴链接 / 公众号文章 / YouTube… 开始分析」+「分析」按钮。点击后自动识别类型（X 帖 / YouTube / GitHub / 网页 / 公众号），把内容送入右侧「快速分析」并直接进入结构化解读对话。
- **今日简报**（置顶卡，底色 `#f4f6f9` + 边框 `rgba(61,90,128,.22)`）：
  - 标题「今日简报 · 7 月 14 日」+ 右侧「查看周报 / 查看月报」（→ reports 页）。
  - **今日焦点**：3 条排行（序号 / 标题 / N 源 / 展开箭头），点击展开显示多个信源版本（来源名 · 类型 · 时间 · 备注）。
  - **选题建议（N）**：默认折叠为一行标题（箭头图标，点击展开）。展开后每条卡片：切入角度（衬线 15.5px）+「角度：… · 时机：…」+「素材：N 条 · 共识/非共识：x/y」+ 三个按钮：查看详情（→ 选题详情弹窗）/ 升级为 Topic / 直接创作 thread（→ 创作页）。
- **排序/过滤行**：排序 综合热度 · 过滤 全部 · 共 30 条（展示性）。
- **信息流卡片**（surface 卡）：标签徽章「已关注」(绿 `#3f7350`/`rgba(63,115,80,.12)`) 或「未标注」(灰) + 作者(600) + 平台(灰) + 右侧「评分 88」(衬线/#3d5a80)。标题衬线 17.5px/600。摘要 14px/1.6 `#4a463d`。标签 chip（`rgba(61,90,128,.09)`/#3d5a80，带小标签图标）。底部动作：选中分析（选中态填充 #3d5a80）/ 跳转原文 ↗ / 仅未关注时显示「＋加为信息源」。

### 2. 素材 notes（素材库）
标题 + 副标题「N 张素材卡片 · 在 AI 对话里点保存到笔记沉淀 · 创作台按段引用」。搜索框 + 按来源/按 Topic 筛选（展示）。卡片：结构化摘录（14px/1.6）+ 底部「来源 《标题》」（回形针图标）+ 明确可点的「跳转原文 ↗」或「查看来源内容」（AI 解读来源）+ 立场徽章（同意/存疑，可选）+ 时间 + 删除图标。

### 3. 信源 sources（信息源登记处）
副标题「登记优质源：内容进 Feed 并高权重排序 · 不是订阅系统」。识别栏（粘贴链接/公众号名 →「识别」）。识别结果卡：可改名输入 + 平台·track_mode 徽章 + 说明文案 + 登记/取消。入口「+ 添加信源池」「+ 批量导入」（→ 弹窗）。已登记列表：名称 + `平台 · track_mode 中文` 徽章 + N 条 + 取消关注。
- **track_mode 四档**（由平台类型自动判定，颜色/文案）：passive 仅标记加权（灰）/ active-rss RSS 轮询（绿 #3f7350）/ active-query 主动查询（蓝 #3d5a80）/ link-only 仅标记跳转（琥珀 #a9791f）。

### 4. 主题 topics（主题库 + 活页详情）
两个视图 `topicView ∈ {list, page}`。
- **列表**：标题「我的主题库（N）」+ 副标题白话「每个主题是一篇 AI 帮你持续维护的综述：存进新素材，它自动更新正文、标出分歧，并记下每次修改」。搜索框（搜已有 + 触发新主题调研）。主题卡：名称 + 状态徽章（持续演进/演进缓慢/持续更新，颜色不同）+「N 条素材 · 综述修订 M 次」+「最新演进：…」+ 可选「⚡ 未解决矛盾」+ 按钮「打开主题 →」「开始创作」。底部「自动建议：新建主题《上下文工程》（来自周报）」。
- **活页详情**：「← 主题库」返回 + 主题名 + 「开始创作」。三块：
  - **主题综述**（AI 维护）：当前认知 / 各方观点（每条含 `[素材N]` 引用，冲突项标「⚡与上冲突」）/ 共识·非共识。
  - **修订记录**（自动生成）：竖向时间线（左侧竖线 + 圆点），每条 日期 + 文本。
  - **待并入素材（N）** + 「全部并入」按钮（点击后 toast「已并入…更新主题综述并记下修订」）。

### 5. 创作 studio（创作台）
「← 返回主题库」+ 标题「创作台」+ 来源标签（Topic/选题）。平台模板分段控件：X thread / 公众号长文 / 口播脚本（切换即重新生成草稿模板）。正文 `textarea`（衬线，min-height 360px）。黄色警示条「段落 4 无素材支撑，创作前请在右侧补充引用」。按钮：重新生成 / 复制全文 / 导出 Markdown。说明：让 AI 按你的意思改在右侧「创作助手」里。
- 草稿导出**只在创作台**（无独立发布页，周报页无导出）。

### 6. 周报/月报 reports（由 feed 内入口进入）
「← 返回 Feed」+ 标题「周报 · …」/「月报 · …」+ 周/月切换。三块：本周动向（↗升温/↘降温，箭头为 SVG）/ 主题更新 / 涌现建议（新建主题建议、矛盾预警、新选题）。

### 7. 设置 settings
LLM Provider（Deepseek 可替换）/ 日报生成时间 / RSS 轮询频率——列表行 + 右侧下拉展示。

## 右侧面板（快速分析 / 创作助手）
状态 `analysisMode ∈ {list, chat}`，可折叠。
- **快速分析（非创作页）**：
  - list 态：显示「已选中（N）」列表（来自信息流卡片「选中分析」，可多选、可移除）+「开始分析 →」；下方提示「分析基于原文；抓取失败会显式标注基于摘要」。无选中且非主题页时显示空态引导。
  - **主题详情页**：list 态改为显示「上下文 · 本主题《…》」+ 快捷问题 chips（核心争议 / 演进脉络 / 适合引用的观点），点击即提问。
  - chat 态：顶部「基于 N 篇内容 · 结构化解读」+ **黄色降级条**「其中 1 篇（公众号）未获取到原文，以下基于摘要」（ADR-007 诚实承诺）。消息气泡：用户（#3d5a80 实底白字）/ AI（`#efece4` 卡，`white-space:pre-wrap`，pending 时三点脉冲动画）。AI 消息下方「保存到笔记 / ✓ 已存入素材库」。底部输入框 + 发送。
- **创作助手（创作页）**：合并了素材与指令改写：已引用（N）列表 / 可插入素材（N，点「插入」把带溯源标记的引用加进草稿）/ 分隔 / 指令对话（「告诉 AI 怎么改…」，如「开头更犀利」「压到 5 条」，AI 直接改写左侧草稿并回执）。

## Interactions & Behavior
- 导航切换页面；左栏与右栏各可折叠（宽度 transition .15s）。
- 今日焦点条目展开/收起多源；选题建议整体折叠/展开。
- 信息流「选中分析」多选累积到右栏；「开始分析」或任何提问都会切到 chat 态并流式出结构化解读（模拟 setTimeout，pending → done）。
- 主题「打开主题」进入活页详情；「开始创作」「直接创作 thread」「进入创作」跳到创作页并预置来源/草稿。
- 保存到笔记：把 AI 回复存为素材卡片 → 底部 toast「已存为素材卡片 · 并入主题《…》并记下 1 条修订」（体现内容飞轮的流转反馈）。
- 关键动作均有底部 toast 反馈（加为信息源 / 登记信源 / 并入素材 / 升级主题 / 跳转原文 / 导出 / 搜索 / 信源池·批量导入确认）。
- 弹窗：信源池（GitHub Trending 等）/ 批量导入（多行识别，按 track_mode 分类，无法识别显式列出）/ 选题详情（角度/为什么现在/共识/非共识/支撑素材 + 升级/创作/忽略）。点遮罩或 × 关闭。
- 对话不落库（刷新即弃），只有「保存到笔记」才沉淀——UI 文案已表达。

## State Management
单组件状态（可拆为路由 + 局部/全局 store）：
- `page`、`topicView`、`activeTopic`、`modal`（pool/import/idea）、`ideaDetail`
- `leftCollapsed`、`rightCollapsed`
- feed：`acquireVal`、`detected`、`expandedFocus`、`ideasOpen`
- 右栏：`selectedItems[]`、`analysisMode`、`chat[]`（{role,pending,done,text,saved}）、`chatInput`
- 创作：`studioPlatform`、`studioSource`、`studioRefs[]`、`draftBody`、`studioChat[]`、`studioInput`
- 信源：`identifyVal`、`identified`、`sources[]`
- 沉淀：`notes[]`；reports：`reportPeriod`；`toast`、`copied`、`assimilated`
真实实现中：内容/来源/素材/主题/报告/草稿建议走后端 API；LLM 调用收敛到统一 provider 层；日/周/月报由定时任务生成（非实时刷新）。

- toast：底 `#211f1a` 字 `#faf8f2`
**字体**：标题/引用 `'Newsreader', serif`；UI `system-ui, -apple-system, sans-serif`；标签/等宽 `'IBM Plex Mono', monospace`。字号：页面标题 23–24px、卡片标题 17.5px、正文 13.5–14px、次要 11–12.5px。
**圆角**：按钮/输入 7–9px；卡片 10–14px；chip/pill 12–22px。
**阴影**：toast `0 8px 24px rgba(0,0,0,.25)`；弹窗 `0 20px 60px rgba(0,0,0,.3)`。
**动画**：`rwUp`（淡入上移 .25s）、`rwPulse`（三点脉冲 1s，延迟 .2/.4s）、折叠 width transition .15s。

## Assets
无位图资源。所有图标均为**内联描边 SVG**（16px，`stroke=currentColor stroke-width≈1.5`，`fill=none`）：资讯=报纸、素材=卡片、信源=人像、主题=文件夹、创作=铅笔、设置=滑块、周报=柱状图；另有回形针、灯泡、时钟、闪电、警示三角、外链箭头、聊天气泡等。品牌处请替换为你代码库里的图标系统。字体用 Google Fonts：Newsreader、IBM Plex Mono。
