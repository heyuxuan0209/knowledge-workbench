# Knowledge Workbench · 知识工作台

> 一个人做内容、为自己搭的开源工作台。**欢迎 ⭐ star、fork 走你要的零件，或一起把飞轮做完。**

**收藏了几百篇，真到动笔却一篇都想不起来？** 收藏不等于拥有。

它的信念很简单：**AI 吃掉一切不产生理解的杂活**——采集、翻译、摘要、聚类、排版、平台适配；**判断、观点、表达，留给人。** 于是你读到的，能真正变成你写出来的、发出去的，再让结果反哺下一次读什么、写什么——一条转起来的**内容飞轮**：

> **读 → 想明白 → 写出来 → 发出去 → 复盘反哺 → 更懂下一篇写什么**

**里程碑**：采集 → 理解 → 沉淀 → 主题 → 灵感 → 创作 → 洞察 已全部产品化；其上又加了**本地语义层**（语义搜索 / RAG / 查重 / 关联，零 API 成本）与**创作台 v2**（文体 × 平台正交拼装）。

> **飞轮还差最后半环**：「发布 → 数据 → 复盘 → 自进化」正在做，目前仍是手动 / 愿景——这也是本项目北极星（内容是否让读者更有收获）真正的落地抓手，**欢迎来认领**。

## 🎯 它解决什么问题

信息过载时代，读得多不等于懂得多。这个系统把「信息 → 认知 → 内容 → 影响力」做成一条流水线：

```
多源信息流入（宽）
    ↓ 采集 + 筛选（AI 全自动：翻译、摘要、聚类、相关性过滤）
主题弹药库（收窄：素材自动归入长期研究主题）
    ↓ 探讨（人的主场：带着问题与 AI 对话，沉淀判断）
深稿（价值凝结点）
    ↓ 裂变（AI 全自动：长文 / X thread / 口播脚本）
中英多平台内容（放宽）
    ↓ 发布 + 复盘
回流进化（闭环：复盘结果反哺信源权重与主题方向）
```

> 目前 **「多源流入 → 深稿」** 已产品化；图中 **「发布 → 复盘 → 回流进化」那半环仍在做**（见开头里程碑）。

分工原则：**AI 吃掉一切不产生理解的环节**（采集、去重、翻译、格式转换、平台适配）；**「决定写什么、核心论点、判断」留在人手上**。

## 🔬 几个可能让你想扒开看的点

- **本地零成本向量层**：bge-m3 跑在 in-process（transformers.js / ONNX），语义搜索 / RAG 问全库 / 查重 / 相关素材关联全本地，不烧一分 API
- **主题是「活」的**：每个主题是 AI 持续维护的综述，新素材自动并入、更新认知、标注矛盾，changelog 就是你的认知演进史
- **文件即行为**：`reference/prompts/` 加一个 md 就多一个平台 / 文体 / 信源——改文件即改产品，贡献门槛极低
- **零依赖后端**：`node:sqlite`，无外部数据库；LLM 只用 Deepseek（¥1/M tokens），能本地算的（主题匹配、热度、动向）都不走 LLM
- **特殊来源智能摄入**：AI HOT 收录页 / 推文先查本地库、公众号直抽正文——不硬抓 SPA 空等
- **火候写作看板**：灵感按「料厚 + 贴合你在养的主题 + 时效」自动分栏，帮你决定"先写哪条"

## ✨ 核心功能（按流水线阶段）

### 📥 资讯 —— 多源采集 + 简报
- **内置源**：AI HOT 精选、Hacker News、GitHub Trending（每日自动同步）
- **登记源**：X 账号（借道 AI HOT 热门转载）、博客/官网（RSS 自动探测，含 6 路径兜底 + 已知无 RSS 站点自动改用社区镜像 feed）、小宇宙播客（免登录追更）、B站 UP 主、YouTube 频道、GitHub 用户、公众号（登记标注）
- **官方源包**：Claude / Anthropic / OpenAI / Google 系官方动态与研究，一键登记
- **简报（都在资讯页）**：日报＝顶部「今日概览」（热点聚类 + 焦点解读 + 选题建议）；点开 **周报 / 月报**＝升温降温动向（本地统计，可展开命中文章验证）、主题演进汇总、涌现建议、深度选题——全板块带溯源链接

### 📝 素材 —— 沉淀
- AI 对话 / 精读里一键「保存到笔记」，自动起标题、自动匹配主题
- **本地语义层**（bge-m3 in-process，零 API 成本）：语义搜索（大白话找料）、把右栏 AI 当搜索引擎问全库（RAG）、自动查重合并、每条素材下「相关素材」关联
- 关键词 / 来源 / 主题筛选搜索（SQL 层，支持多关键词模糊匹配）

### 📚 主题 —— 知识加工（核心资产）
- 每个主题是一篇 **AI 持续维护的综述**：新素材自动并入，更新认知、标注观点冲突
- 匹配可解释：每条素材显示「因共享『xx』『yy』被匹配 + 相似度」，误并一键移出并修订综述
- **探讨**：主题右栏与 AI 对话，综述 + 已并入素材做弹药（引证据、指矛盾、提反例）
- changelog 时间线 = 认知演进史

### 💡 灵感 —— 选题种子
- 选题种子收集箱，和素材分工：**素材是「料」，灵感是「要写什么」**（两者用支撑素材挂钩）
- **即时分析入口**（在灵感页）：任意链接 / 大段文字 / 音频 / PDF 丢进来 → 结构化中文精读，产物落素材、可一键提为灵感；多模态（视频字幕→ASR 三级降级、播客转写、说话人分离 faster-whisper + pyannote）；**特殊来源智能处理**（AI HOT 收录页·推文先查本地库不空抓、公众号直抽正文）
- 三种收录：随手记一句 / 资讯卡一键收进 / 即时分析产物提为灵感；AI 从周报涌现的选题也在此统一收口
- **收集视图 / 写作看板** 双镜头：看板的列由系统按「火候」（料厚 + 贴合你在养的主题 + 时效）自动判定——攒着 / 可以写了 / 在写，帮你决定"先写哪条"
- 手记灵感自动语义挂靠相关素材（作建议，点采纳才算料、才影响火候）
- 外部灵感源通用接入口（飞书妙记 / 纪要 / 群聊 / 云文档 收件箱——已留占位，待接凭证）

### ✍️ 创作 —— 内容生产
- **文体 × 平台形态正交拼装（创作台 v2）**：文体与平台两轴独立组合，选素材 → 系统按素材荐文体 + 你选平台 → **只用选中素材**起稿；平台模板即文件（`reference/prompts/creation/` 加 md 即扩展）
- 从主题 / 灵感 / 素材起稿；段落级溯源（[素材N] 标记落库，导出时转参考来源列表）
- 小红书**图文卡片**：文字稿一键渲染成图卡（HTML 渲染，多风格，本地导出 PNG）
- **平台裂变**：写好的稿子点其他平台按钮，改写为该平台规格版（新稿另存，原稿不动）
- 修改工具包：**审稿**（挑剔读者/结构编辑/事实核查三视角批注，逐条可应用可撤销）、**3 个改法**（选段给锋利/具体/简洁三个候选）、润色降 AI 味、标题候选、指令改写

## 🚀 快速开始

```bash
# 1. 安装依赖
cd backend && npm install
cd ../frontend && npm install

# 2. 配置环境变量
cd ../backend
cp .env.example .env
# 填入 DEEPSEEK_API_KEY（https://platform.deepseek.com）
# 可选：HF_TOKEN（说话人分离用，Hugging Face token）

# 3. 初始化数据库（基准 schema 即最新结构）
mkdir -p data
sqlite3 data/app.db < src/db/schema-v3.sql

# 4. 拉取首批内容
node src/services/sync-aihot.js

# 5. 启动
node src/server.js                 # 后端 :3000（Terminal 1）
cd ../frontend && npm run dev      # 前端 :5173（Terminal 2）
```

访问 http://localhost:5173/

**定时化（可选）**：`sync-aihot / sync-hackernews / sync-rss / sync-github-trending / sync-active-query / sync-daily-report / sync-period-report` 均为独立可执行脚本，按需挂 crontab（早晚同步 + 每日简报 + 周一周报）。

## 🛠 技术栈与成本原则

| 层 | 选型 |
|---|---|
| 后端 | Node.js 26 + Express + node:sqlite（零外部数据库） |
| 前端 | React 18 + Vite（手写 workbench.css 设计系统） |
| LLM | Deepseek（¥1/M tokens，OpenAI SDK 兼容），SSE 流式 |
| 多模态 | faster-whisper（本地转写）+ pyannote 3.1（说话人分离）+ yt-dlp / bili-cli / gh |
| 验证 | Playwright 驱动真实 UI 的端到端验证 |

**成本原则：本地零成本优先。** 主题匹配（TF 余弦）、热度统计、关键词动向全部本地计算不走 LLM；LLM 只用在翻译、摘要、同化、起稿等真正需要语言能力的环节，且批量合并调用。
