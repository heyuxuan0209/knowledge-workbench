# link2article — 链接 → 中文结构化文章

## 工作流(两步)

### 第 1 步:生成素材文件(脚本干)

```bash
python3 scripts/link2article.py <链接>
```

- 自动按域名分流:YouTube(优先抓字幕,无字幕才转写)、小宇宙(解析音频直链→转写,带上官方 shownotes)、抖音(yt-dlp 尝试,失败则提示用户先用去水印工具拿文件)、音频直链、其余按网页提正文。
- 产出 `output/<标题>.material.md`:含 YAML 元数据 + 原始正文/转写稿 + 时间戳 SRT。
- 常用参数:`--whisper-model small`(更准)、`--no-asr`(只要字幕/正文)、`--file 本地文件`(用户自己下好的音视频)。
- 首次转写会下载 whisper 模型;依赖:`pip install faster-whisper trafilatura yt-dlp --break-system-packages`,需要 ffmpeg。

### 第 2 步:翻译 + 总结(你干,不要调其他工具)

读取生成的 `.material.md`,注意 frontmatter 里的 `language_hint`:

- 若标注"需翻译":先完整理解原文,再**用地道的中文改写**(不是逐句直译;术语保留英文原词并括号加中文)。
- 若已是中文:跳过翻译,直接总结。

然后输出一篇**中文结构化文章**,存到 `output/<标题>.article.md`,固定模板:

```markdown
# {吸引人的中文标题,不超过 25 字}

> **TL;DR**:3 句话以内讲清这条内容的核心。

**来源**:{source_url} | **类型**:{视频/播客/文章} | **时长/篇幅**:{...}

## 核心观点

{3-6 个小标题分段,每段 2-4 句,覆盖内容主干。观点为主,过程为辅。}

## 金句摘录

{2-4 条最有传播力的原话,中文呈现;外语内容附原文。有时间戳的标注时间点。}

## 可复用素材

- **选题角度**:{这条内容可以衍生出哪 1-3 个自媒体选题}
- **数据/案例**:{可引用的具体数字、案例、故事}
- **争议点/评论区钩子**:{适合引发讨论的点}

## 局限与存疑

{内容里站不住/需要核实的说法,1-3 条;没有就写"无明显问题"}
```

## 规则

- 金句必须来自素材原文,不得编造;时间戳来自 SRT。
- "可复用素材"要贴合用户的自媒体场景(短视频/图文选题)。
- 素材超长(>3万字)时先分段读完再总结,不要只看开头。
- 转写稿常有错字(同音字),总结时按上下文纠正,金句处标注"(转写稿,字词或有出入)"。
