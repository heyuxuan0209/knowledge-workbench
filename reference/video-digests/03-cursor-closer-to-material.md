# 贴近材料：当 AI 让执行变得廉价，人的位置在哪里

> 原题 "Closer to the Material: How AI Changes How We Build and What It Must Not Erase" | Ryo Lu（Cursor 设计负责人，曾任职 Notion）| Compile 26 大会演讲 | https://www.youtube.com/watch?v=az6OEZV8iHw

**一句话核心**：AI 把"造东西的循环"从数周压缩到数秒，但如果界面把循环藏进黑箱，人就退化成"批准者"（approver）；好的 AI 工具应像"玻璃"（Glass）一样让过程可见、可介入，让人始终是作品的"作者"（author），因为品味和判断只在循环内部形成。

## 讲述脉络

### 一、开场：Ryo OS——一个"能在里面思考的地方"

这场演讲本身写在 Ryo OS 里——Ryo 用 Cursor 给自己搭的一个小计算环境。动机是怀念一种感觉：计算机曾像开放的游乐场（open playground），不是孤立的应用，也不是命令行，而是可以随手摆弄、在其中思考的地方。放在过去，这种东西只会停留在脑子里：一条笔记、一张草图，永远不会成真。但现在循环（loop）不同了：他可以先造出一小块，玩一玩，发现哪里不对，让 Cursor 重塑、演化、改细节，一遍遍地试。最终 Ryo OS 从一个"vibe-coding 出来的音效板"变成了一个能在里面思考的地方——"它有我的灵魂"。

这就是当下的特殊性：AI 正在坍缩想法与现实的距离。你可以打一段字看它活起来，可以让智能体（agent）读完整个代码库、制定计划、编排更多智能体、验证正确性、持续循环。但这让老问题更尖锐：当更多东西能被更快造出来，问题不再是"能不能造"，而是"什么应该存在"；更个人地说：**这个工具是把我带向真相，还是推得更远？**

### 二、AI 改变的最重要的东西：循环本身

以前做软件是慢循环：想法→规格文档（spec）→设计师设计→工程师实现→PM 评审→QA 测试→下一件事。循环昂贵，所以我们严防死守：动手前先争论、先规划，努力"提前正确"，因为犯错太贵。

现在循环快得多：有想法，要一个版本，检查、改、跑、继续——以分钟甚至秒计。Ryo 强调这不只是生产力变化，**它改变了判断力（judgment）形成的方式**。第一版几乎从来不对：概念太模糊、过渡动画发飘、间距数值正确但视觉上不对（optically wrong）、文案说了"正确的话"却不是"真实的话"——这些只有东西存在之后才能察觉。**那个小小的修正动作正是判断力形成之处**：你尝试，世界回应，你察觉不对，你调整，积累直觉。品味（taste）在这个循环里形成，手艺（craft）在这里发育。

所以 AI 的深层承诺不是"更快造更多"，而是**让更多人以自己想要的深度进入这个循环**：设计师更贴近实现，工程师探索不同架构方向，从未自认是程序员的人造出想象之物的第一版。

### 三、危险：循环被藏起来，人变成"批准者"

但循环变快时也可能被藏起来，一旦被藏，人就不再从中学习。工具变成黑箱（black box）：许愿、等待、拿到输出、接受或拒绝。成功不知为何，失败不知在哪，它改变了你的工作方式你也察觉不到。这不是建造，是拉老虎机。循环之所以重要，是它让你保持清醒、逼你注意、要你在乎。**如果 AI 让执行更便宜却让判断更弱，我们得到的不是文艺复兴，而是"带着能用按钮的垃圾"（slop with working buttons）**。

这个时代两件事同时为真：更多人能造，更多平庸之物也能被造。代码更多、原型更多、功能更多，但流向决策的注意力可能更少——什么该存在？为什么？什么该被拒绝？哪里该慢下来？当制造成本趋近于零，平庸的东西并不显得糟糕：能跑、能发布、填满空间，而且可以有无限多个——能运转但毫无意义的软件、正确但死气沉沉的界面、由智能体制造、无人在乎、发射进虚空的产品。

真正的风险不是机器太有创造力，而是**人在机器周围变得太被动**。危险的版本非常舒适：你许愿，机器消失，回来时带着看似合理的东西——按钮能按、demo 能过——但你从未身处决策之内，只是在批准表面。危险在于**界面把人变成批准者**，在于我们混淆了输出（output）与材料（material）：**输出终结循环，材料把你请回循环**。输出说"这是答案"，材料说"摸摸它，塑造它，把它变成你的"。这个区别至关重要，因为制作（making）不只是产出工件，**制作是一种思考方式**——AI 若把过程藏得太深，仍能给你输出，却拿走了让你变强的挣扎。Ryo 不想要软件自己建造、人类旁观的未来，那太无聊。

### 四、Glass：Cursor 的界面原则

Ryo 因切身感到这个问题而加入 Cursor：让工具灵活到从最简单的需求伸缩到最专业的场景，而非把所有人变成程序员。Cursor 继承自 VS Code 曾是巨大优势，但世界变化太快——模型更强、智能体更能干、交互形态在漂移。问题不再是"怎么在编辑器旁边加 AI"，而是：**如果今天在智能体的世界里从头设计一个软件环境，它该是什么感觉？**不是 IDE 挂一个聊天面板，而是一个**人和智能体在同一份材料上共同工作**的地方。

这就是新界面（内部代号 Glass）的动机。Ryo 澄清：Glass 不是视觉风格，与毛玻璃模糊或"液态玻璃"无关，**它是界面原则：系统应让你看穿它的工作**——不是你要控制一切，而是有时那一个小地方就能告诉你整体对不对。黑箱里 AI 消失后返回裁决（verdict）；Glass 里你看得到计划、思考和工具调用的流式呈现、改动、执行的命令，可以叫停、塑形、深入检查、接管。**你不必读每一行，但永远可以读**。这个选择权服务三种人：老手放手让智能体跑、需要时跳进去；设计师接受结构、只调那个让感觉对起来的细节；新手放慢速度、读计划、通过看见来学习。同一个系统支撑三者——**渐进披露（progressive disclosure），无限深度，选择权在用户**。

AI 越强这越重要：智能体工作以分钟计时，你需要操纵（steering）；以小时计时，你需要整个系统的可读性（legibility）——知道发生了什么、检查状态、设计护栏（guardrails）、表达你想要什么。被隐藏时你只是在许愿，而"许愿式思维不是好界面"。**没有可见性的力量不是工具，是风险**。答案不是拒绝自动化（太蠢），也不是永远手动介入每一步（太累），**而是选择**：能走出机器让它承载工作，也能在"贴近"重要的时刻走回去——因为有些决定不只是任务，它们是你的品味进入材料、价值观变得可见、作品开始"像出自某人之手"的地方。黑箱为干净输出优化，Glass 为人的能动性（human agency）优化；黑箱把软件简化成一个愿望加一个裁决，Glass 把软件变成黏土（clay）。**Glass 是一种贴近材料的权利**。

### 五、方法论：Baby Cursor 3 到 Baby Glass

今年一月，团队开始这场更激进的界面重置。Glass 起点叫 Baby Cursor 3：基于 Cursor CLI 的 Electron 原型，重点不是复刻生产架构，而是造一个能"用手思考"、没有 VS Code 包袱的快速游乐场，同时探索大结构与小机制：界面服务"一到 N 个智能体、零到 N 个项目"时该是什么感觉？默认可见多少状态？在不同概念和作用域（scope）间导航是什么体验？**这些问题在文档或 Figma 稿里答不了，必须造出来、用起来、察觉不对、再改**。

因为是"用 Cursor 造 Cursor"，循环极紧：想法、原型、反应、修改、重复。Ryo 说速度本身不是重点，重点是**更快的原型让团队更快抵达确信（conviction）**：不靠文档和观点对齐人，而是通过 demo 看见；不隔着距离预测产品，而是通过建造发现产品。某个时刻原型不再是实验，他们把它变成了真的。之后演化为 Baby Glass：新原型环境改为 Web 应用——这很关键，因为**分享变容易了**，更多人能在同一个游乐场工作，原型从私人沙盒变成"共同思考未来的共享环境"。这就是他想要 AI 工具创造的循环：不只是更快的输出，而是更快地接触材料、学习、产生分歧、协作、形成确信——因为最好的想法从不完整降临，只在你贴得足够近时逐渐清晰。

### 六、循环之前与之后：智能体拿不走的两端

如果智能体能执行越来越多的循环，留给人的是**循环之前和之后**。

**之前，是让你开始的东西**：一个无法忽略的摩擦、一个困扰你的缺口、一种"某个东西应该存在"的感觉。他以自己为例（现场插播了一段早年苹果发布会片段，谈 Mac 界面的双重目标：给专业用户更强大的界面，同时成为从未碰过电脑的人的梦想界面）：曾经用 Mac 像触摸活物——Dock 弹跳、精灵效果（Genie effect）俯冲、Exposé 把窗口像纸牌摊在桌上。都不必要，但都让人感到有人在乎；软件有质感（texture）、有观点，你能感觉到背后的人。后来软件更顺滑、更一致，却更无趣：怪癖因测试表现不好被删，温度因不可度量被砍，"我们把自己优化进了一个一切完美运转却毫无感觉的世界"。现在 AI 让速度非人类——一下午生成一个产品、一杯咖啡前跑出十个变体——但陷阱相同：**当造东西几乎免费，垃圾（slop）也免费了**。这正是人更重要的原因：**品味不是提示词，在乎不是参数**，你放进作品的那种怪异、具体、个人的东西无法被平均出来。他在 Notion 在乎的是给人能变成其工作形状的简单原语（primitives）而非又一个单用途应用；在 Cursor 在乎的是给人一个**能始终做自己作品作者的环境**。Ryo OS 是同一直觉的个人化形态：如果文件、应用、音乐、笔记、设置、记忆、智能体都是同一种东西呢？如果 demo 不是一页"软件正在变成黏土"的幻灯片，而 demo 本身就是那块黏土呢？**这种最初的痒（itch），智能体无法替你生成——只能来自活在世界里、注意到哪里不对劲**。

**之后，是责任（responsibility）**：我们向世界释放了什么？改变了什么？服务了谁？让什么变得更容易成为现实？智能体能执行、倍增循环，做出 1000 个版本，但它们不知道什么值得保护，不知道一个东西是否"技术上正确但感觉是死的"，不知道它正把我们悄悄拉向什么未来——这些仍在我们身上。由此得出：**手艺不消失，它会移动**。执行昂贵时，手艺活在执行本身——精心的代码、精确的布局、改写 12 遍的小文案；生成廉价后，手艺**上移到判断**（问什么、留什么、拒绝什么、在哪慢下来、不做什么），**下移到责任**（发布了什么、改变了什么、服务了谁）。制作不只是产出工件，**是对"什么样的世界应该存在"下注**。

### 七、收尾：为什么而造

演讲以 Ryo OS 里的一条笔记开始，结束时笔记变了——不是因为想法变完美，而是它变真实了，成了一个可以在里面思考的地方："我可以用这个东西来思考这个东西本身"。这就是他建造的原因：不为生产力、不为颠覆，而是**脑中之物变得真实到别人能看见、能触摸的那一刻**。工具、模型、经济都会变，但这部分不变：你感觉不对，落下一笔，世界回应，你把它做成真的，最终它真实到触动别人。他要的是让循环更短、更清晰、更诚实的工具，让人留在工作中——不是被动的批准者，而是作者。**AI 的意义不是让人消失，而是让更多人感受到"我觉得这应该存在，而且我能把它做出来"。做对了，未来不只是更快，而是更人性**。

## 关键案例与细节

- **Ryo OS**：Ryo 用 Cursor 造的个人计算环境，演讲稿写在里面；从"vibe-coding 的音效板"进化为"能在里面思考的地方"，是"想法→现实距离坍缩"和"demo 本身就是黏土"的活案例。
- **判断力形成的四个信号**：概念模糊、过渡发飘、间距"数值正确但视觉上错"、文案"正确但不真实"——都只有在东西存在后才能察觉。
- **黑箱 = 老虎机**：许愿→等待→输出→接受/拒绝；成功不知为何，失败不知在哪。
- **Glass 的三类用户**：老手放手+随时接管；设计师接受结构+微调关键细节；新手放慢+读计划+看着学。原则：渐进披露、无限深度、选择权归用户。
- **时间尺度对应界面需求**：智能体跑几分钟→需要 steering；跑几小时→需要整个系统的 legibility + 状态检查 + 护栏设计。
- **Baby Cursor 3 → Baby Glass**：一月启动；前者是基于 Cursor CLI 的 Electron 原型（甩掉 VS Code 包袱），后者改为 Web 应用，只为分享容易——原型从私人沙盒变成团队共享的思考环境；靠 demo 而非文档达成 conviction。
- **Mac 质感清单**：弹跳的 Dock、Genie 效果、Exposé——"都不必要，但都说明有人在乎"；后来 quirks 因测试数据差被删、温度因不可度量被砍。
- **手艺迁移公式**：执行贵→手艺在执行里（改 12 遍的文案）；生成廉价→上移到判断、下移到责任。

## 值得记住的表述

- "Taste is not a prompt. Caring is not a parameter."——品味不是提示词，在乎不是参数。
- "Output ends the loop. Material invites you back in."——输出终结循环，材料把你请回循环。

## 给 Jessica 的 idea 钩子

1. **按 Glass 原则设计工作台的 AI 功能**：不要让 AI 只吐结果，而要让计划、中间推理、改动可见、可打断、可接管。三栏布局天然适合"渐进披露、无限深度"——比如中栏放产出，侧栏放默认收起但"永远可以读"的 agent 过程层。这可能就是你在找的答案：AI 时代的界面不是聊天框，而是人和 agent 在同一份材料上工作的环境。
2. **警惕 AI 把你的写作循环变成黑箱**："制作是一种思考方式"、判断力只在修正中形成——这正是你"通过写作加深理解"的理论依据。把 AI 产出当 material 而非 output：草稿始终可揉捏（分块改写、保留痕迹、多版本对照），而不是"接受/拒绝"两个按钮，否则中英双线高产出容易滑向内容版的"slop with working buttons"。
3. **"筛选"和"复盘"正是手艺迁移的落点**：生成廉价后，手艺上移到判断（选什么题、拒绝什么）、下移到责任（发出去的东西改变了什么、服务了谁）。可在工作台把这两端做成一等公民——如"拒绝清单"（记录不写的理由）和结构化复盘视图——这是 agent 拿不走的护城河。
4. **用 Baby Glass 的方法迭代工作台本身**：界面问题"在文档或 Figma 里答不了，必须造出来感受"。与其规划完美布局，不如维护一个轻量原型分支，快速试"默认可见多少状态""在不同 scope 间导航是什么感觉"，用可运行的 demo 达成自己的 conviction——这个过程本身就是很好的 build-in-public 内容。
5. **终极目标对标 Ryo OS："能在里面思考的地方"**：如果信息、笔记、草稿、agent、复盘都成为同一种可揉捏的材料，工作台就从流水线进化为思考场所，"用这个东西来思考这个东西本身"——这个进化过程正是你 AI 产品灵感最直接的来源。
