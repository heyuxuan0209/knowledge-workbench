# 代码泛滥、市场稀缺：AI 时代产品经理的新工作说明书

> 原题：The New PM | 演讲者：Claire Vo（ChatPRD 创始人，二十余年产品/设计/创业老兵）| 场合：Compile 26 大会（Cursor 社区）| 视频：https://www.youtube.com/watch?v=4CAFK-rc26A

**一句话核心**：当 AI 让"造东西"不再稀缺，产品工作的瓶颈从工程产能转移到市场需求——新一代产品人的三件事是：找到钱（可商业化的想法）、从第一性原理构想全新产品、并说服人们跟随你的愿景。

## 讲述脉络

### 一、老格言与老毛病：我们一直在发明"不造东西"的花样

Claire 开场把讨论对象放宽：她说的"产品经理"泛指产品人、产品 Agent、"产品自治循环"——总之是决定"造什么、为什么造、怎么造"的角色。产品界的第一格言是"build something people want"（造人们想要的东西），但她点出尴尬事实：**这恰恰不是我们花最多时间做的事**——几十年来，行业一直在发明越来越有创意的方式来"不造东西"。

她列举了一串"受害者都认识"的图表：敏捷（Agile）的循环与阶段闸门，过不了关就打回积压列表（backlog）；backlog 本身——一堆不确定有没有产能去做的想法，要进会议、在表格里辩论数字、再排进冲刺（sprint）；PRD——连按钮怎么工作都要写清楚的文档，因为十年前工程资源金贵到工程师不被允许自己决定按钮的行为；还有 SAFe（规模化敏捷框架），她调侃"不知道 SAFe 的人真是没吃过苦"。

关键转折：她把话锋指向当下——**多 Agent 循环、自主编码（autonomous coding）、"token 榨到最满"（token maxing），本质上是同一种冲动的新表达**。这些精巧系统只是"工作如何完成"的描述符，和敏捷循环、backlog 流程图没有本质区别：我们离"造人们想要的东西"依然很远，于是发明各种"感觉很有生产力"的事来做，而且是在还不知道产出是否有价值之前就做。她坦承自己也"交 token 税"。她现场投票：谁写的代码比以往任何时候都多？（一片举手）谁的收入随 PR 数量成比例增长？（没人举手，Cursor 员工被要求回避）——这就是问题所在。

她也承认，两年前"不造东西"是合理的：建造昂贵、缓慢、有风险，产品经理的职业几乎就是学会说"不"——她两年前的演讲提到，"产品经理如何学会说 no"有近 9.99 亿条搜索结果。

### 二、约束被打破：从"保护工程师"到"产能不再是问题"

旧系统存在的原因是工程师是超稀缺资源。产品经理因此成为"多疑的守门人"：说不、砍范围（scope cut）、保护宝贵的工程产能，整个公司都练就了一套对想法保持怀疑的漏斗。

现在这个漏斗在产能意义上不需要存在了。她引用 Anthropic 的图表：**工程师每季度交付的代码量是过去的 8 倍**——度量未必完美，但趋势无可否认。个人和团队的交付能力都在飙升，即将超过那个"好点子积压列表"的规模。约束转移了，无论工程、产品还是创始人，工作方式都必须变。

但另一张大家都见过的图表揭示了裂缝：**App Store 提交量和上架应用数在涨，但安装量、评分和使用量没有涨**。生产在爆炸，需求却不成比例。她总结这个时代的"丰裕"：token 丰裕、智能丰裕、代码丰裕，更多建造者（包括非技术人群）、更多产出——多到"我们需要两个 GitHub"（她引用当天大会上听到的说法）。**但"人们想要的东西"没有变多，"人们愿意付钱的东西"更没有变多。**

这就是她定义的时代级产品问题、也是商业问题：**产能没有地方可以着陆**——其中有技能问题（一整代产品人被训练成守门人）、框架问题和市场缺口问题。好在，产品人最爱的就是问题陈述。

### 三、新产品工作的定义：不是管产能，也不是做原型

Claire 给出她的答案。新产品工作**不是**思考工程产能（"那是个很无聊的工作"），也**不是**做原型（prototyping）——她称之为"偷懒的过渡步骤"，用来给自己升级技能还行，但不是产品工作的核心。核心是：**找到方向、决定什么值得造、并证明它能产生影响**。她特意强调自己喜欢 ARR（经常性收入）——产品要落到真实的商业。

她把"人们想要的东西"拆解成两个可操作的判据：

**1. 商业价值（commercial value）：人们会为解决这个问题付钱吗？** 她区分了"个人软件"和产品市场契合（product-market fit）：她给自己做过一个 64 首歌的 emo 金曲对决 App"March Sadness"，那是"她想要的东西"，但市场只有她一个人。谈钱有点"不体面"，所以很少有产品人把这个问题放在第一位，但她的假设是：**这个时代的限速器（rate limiter）不是智能，而是可商业化的想法**——不是"我能不能发布这个功能、有没有人用"，而是"发布之后会不会发生以金钱为结果的价值交换"。钱不会自动跟着功能来。

**2. 行为改变（behavior change）：你能让人们改变行为来采用它吗？** 这是她看到大多数人失败的地方。识别问题容易（"Claire 需要选出最好的 emo 歌"），造出方案也容易（"vibe code 一个对决 App"），但你能让 100 个、1000 个人登录投票吗？在功能可以"从 IDE 里飞出来"的时代，这一环被严重低估。她近二十年的经验是："发布了、有合理价值，人们自然会用"——**这几乎从来不成立**；发布后真的发生行为改变，近乎魔法。在体验、定位（positioning）、分发（distribution）上反复拧螺丝以促成行为改变，是产品最难的工作之一，但没人爱谈——大家谈的都是"要是有这个东西该多酷"，而不是"谁会真的改变工作或生活方式来用它"。

她说：如果你作为产品经理只做两件事，就做这一页 PPT 上的两件。

### 四、AI 帮不上忙的事：从第一性原理构想全新想法

接下来是她认为 **AI（至少目前）无法代劳**的一环：从第一性原理（first principles）孕育新颖想法。她非常努力地尝试过和 AI 头脑风暴：AI 能把已有想法打磨得更好，也能相当有效地探索邻近空间（adjacencies），**但不擅长构想一个"尚无现实依据"的未来世界状态**——一个你相信存在、能通过行为改变去实现、且有商业价值的未来。

由此推论：**抄产品不再是优势，发布更快也不再是优势**。当人人都能快速复制，真正的产品经理工作变成：拥有独特的世界观，能清晰描述那个产品的形状，并在没有任何证据的情况下、带着强烈信念把它造出来。她认为产品人的这块肌肉已经萎缩太久——因为建造昂贵，大家习惯了"渐进式、有证据才挤一点工程产能"，而不是"大胆地、无证据地发布"。

她以经历背书：做过一家实验（A/B 测试）创业公司，"基本在世界上每一家 A/B 测试公司都工作过"，结论是——**有些事你就是无法通过数据知道**。没有哪个模型会替你确认全新想法能在市场成功（"当然你可以挑个嘴甜的模型，它会说你是天才"）。信念（conviction）与愿景（vision），是产能不再受限时代产品人的本职。

### 五、分发：把人们带上路

第三件事是分发。产品不止于"问题+解决方案"——发现问题、造出方案人人都会，稀缺的是**把人带上路（bringing people along）**：告诉世界"我认为世界会往这里走，请把你的马车挂到我的马车上"。对象不只是客户和社区，还有队友、员工，乃至**整个市场**——市场跟随你的愿景，你才能"资本化"（她强调这个双关）眼前的机会。

这仍然是人类的工作。她抛出自己的名言"Rizz is the only moat"（个人魅力是唯一的护城河）：当任何人都能写代码、能发布、能搭 Agent、"很快任何人都能把数据中心放上月球"，护城河到底是什么？**是谁能说服市场上最好的人才、说服客户、说服整个市场跟随并投资于你的愿景。** 这项技能需要刻意练习，既要对内（团队）也要对外（市场）。

小结：把"人们想要"拆成商业价值+行为改变、在缺乏数据时从第一性原理发展全新想法、并激励人们跟随——这些才是让海量代码产生杠杆、配得上"精美流程图"待遇的东西。行业不谈它们，因为它们稀有、艰难、充满失败，但这就是好产品管理和伟大公司的真相。

### 六、对个人意味着什么：头衔会活下来，工作不会

她回应大家真正关心的问题："PM 死没死？能不能裁掉？工程师会不会取代 PM？"她几年前预言 AI 会"压扁产品职能栈"（collapse the product stack）：能做这份工作的人可以来自任何职能——愿意升级技能的传统 PM、工程师（"说工程师没有产品思维纯属胡扯，'你不配决定按钮'的时代结束了"）、当然也包括设计师。她仍相信角色边界会塌缩。

更新后的判断是：**头衔（title）会存活——组织和公司对头衔很依恋——但工作（job）不会**。PM 的工作会变得非常不同；工程师的工作已经不同；设计师的工作更是——她自己当设计师时还在 Photoshop 里"手工艺式地磨圆角、手画阴影"。她给出一个警告式的判断：如果你的工作和一两年前感觉没有区别、也没有越来越聚焦于上述这些事，**你最终会得到一个非常大的代码库和一个非常小的公司**——这是所有人都要避免的结局。

### 七、什么不变：用户、瞄准，以及钱

当所有职业身份都被压缩成桌面上的图标（她借机把自己的 ChatPRD 图标放在估值 600 亿美元的 Cursor 图标旁边），剩下什么？**用户。** 行业谈了太多代码、基础设施、"怎么造"，谈了一点"造什么"，但几乎不谈"谁在消费、为什么"。她希望把人类和 AI 的全部智能对准：人类行为改变、商业价值、世上应该存在的新事物、以及让人们相信愿景。

她澄清自己不是反速度派——她的名言之一是"fast beats right"（快胜过对），快速进入市场几乎是产品工具箱里最有价值的东西；演讲期间估计已有 13 个 PR 提交待审。**但你得知道你的"slop cannon"（烂活大炮）往哪儿瞄**。她自己正压着 64 个 PR：有些是好想法，有些"造了都忘了"，有些确实该合并——风险在于形成不落地于市场的空转（churn）和 token 消耗。

结论：你可以把 AI 当**盾牌**用——就像过去把流程当盾牌——用它发代码、谈论"活动"、发明花哨的建造方式，都行；但更应该把 AI 当**加速器**：加速愿景 → 市场影响 → 以及她"最爱且很不含蓄"的——钱。做这一切是为了造伟大的企业，不只是伟大的产品。

她的宣言收尾：**找到钱，构想全新想法，把人们带上路。这就是"人人都能造"时代的产品工作。别只写代码——去建公司。**

## 关键案例与细节

- **8 倍**：Anthropic 图表显示工程师每季度交付代码量为过去的 8 倍——产能约束被打破的核心证据。
- **App Store 剪刀差**：应用提交量/上架量上升，安装、评分、使用量持平——"生产爆炸、需求不涨"的市场侧证据。
- **9.99 亿条搜索结果**："产品经理如何说 no"——旧时代 PM 以拒绝为业的注脚。
- **"March Sadness" 64 首 emo 歌对决 App**：她自己 vibe code 的个人软件，用来区分"某人想要"与产品市场契合——市场只有她一个人。
- **64 个待处理 PR**：她自己的仓库现状，具象化"slop cannon 需要瞄准"——好想法、忘了为何而建的东西、该直接合并的东西混在一起。
- **现场双问投票法**："谁写的代码更多了？"（全场举手）→"谁的收入随 PR 成比例增长？"（无人举手）——一个可复用的自检问题。
- **AI 头脑风暴的能力边界**（她的实测）：能优化已有想法、能探索邻近空间；不能凭空构想无现实依据的未来状态。
- **A/B 测试老兵的反数据宣言**：做过实验创业公司、遍历各大 A/B 测试公司后的结论——有些事无法靠数据得知，必须靠信念下注。
- **判据二分法**：商业价值（会付钱吗）× 行为改变（会改变习惯来用吗）——两者都过关才算"人们想要"。

## 值得记住的表述

- "Rizz is the only moat."——个人魅力（说服力）是唯一的护城河。当人人都能写代码、造 Agent，能说服顶尖人才、客户和整个市场跟随你的人才有壁垒。
- "Don't just write code. Build companies."——别只写代码，去建公司。整场演讲的落点：产出要落在商业上。

其余金句转述：她还说过"快胜过对"（fast beats right），但前提是知道往哪儿瞄；她警告"不进化的人会得到一个很大的代码库和一个很小的公司"；她断言头衔会活下来、但工作不会。

## 给 Jessica 的 idea 钩子

1. **给知识工作台加一道"落地判据"**：Claire 的核心自检是"产出更多了，被消费/付费的部分涨了吗？"你的六步流程很容易变成她说的"感觉很有生产力的系统"。可在复盘环节固化两个字段：这篇内容/这个功能促成了谁的**行为改变**（订阅、回复、复访）？离**价值交换**（付费、合作、线索）近了一步吗？——防止工作台沦为自我感动的敏捷图。
2. **内容创作就是在修"唯一的护城河"**：按"Rizz is the only moat"的逻辑，你的中英双线写作不是 AI 产品开发的副业，而是主业的一半——它就是"把人们带上路"的分发肌肉。可以有意识地把每篇内容当成"愿景叙事"练习：不只解释一个技术点，而是陈述"我认为世界会往这里走"，让读者提前挂上你的马车，为未来产品预置市场。
3. **AI 分工的明确边界，直接可用于你的创作流**：Claire 实测的结论——AI 擅长打磨已有想法、探索邻近空间，不擅长从第一性原理构想无先例的未来。对应到你的工作台：探讨沉淀环节让 AI 做"邻近探索"（相关案例、反方观点、类比领域），但选题立意和产品愿景这两步刻意不外包，用写作本身来练"无证据下的信念"这块肌肉。
4. **独立开发者的产品筛选器**：你就是她说的"压扁的产品职能栈"本人——一人身兼 PM/工程/设计。在动手 vibe code 任何新工具前，先过她的二分法：这是"March Sadness"（只有我想要的个人软件，做了愉悦自己即可）还是有真实付费意愿的问题？如果是后者，先想清楚"谁会改变现有工作习惯来用它"，再写第一行代码——因为写代码已经是最便宜的一步。
5. **"瞄准 slop cannon"的双语内容版**：她压着 64 个 PR 的窘境在内容创作里同样成立——AI 让多平台双语产出极便宜，稀缺的是真实的消费与互动。与其追求发布矩阵覆盖率，不如按"安装量而非提交量"的思路砍掉无人消费的渠道，把产能投给能验证行为改变的一两个阵地。
