# 当 AI 接管了写字，写作者去哪了：Anthropic 内容设计负责人的三次自我重构

> 原题：Writing for humans in an AI world | 演讲者：Chelsea Larsson（Anthropic 内容设计 / Content Design 负责人）| 场合：Figma Config 2026 | 视频：https://www.youtube.com/watch?v=l5VRhrNeidY

**一句话核心**：当 AI 把"生成文字"自动化之后，写作者的价值不降反升——语言已从"设计的说明"变成"设计本身"，人的工作是向上游走：塑造 AI 的语言层、为用户弥合能力鸿沟、并把"人的位置"写进未来的设计默认值。

## 讲述脉络

### 开场：一个"人肉文本生成器"的存在危机

Larsson 自嘲是"人肉文本生成器"：几年前 AI 走红，大家最先自动化的就是她的活——标题、报错信息、UI 文案。她真心怀疑"我们是不是完蛋了（are we cooked）"。讽刺的是，她随后被 Anthropic 聘去领导她以为正被 AI 消灭的内容设计岗位。入职一年半，结论：没完蛋，但一切都变了。

例子：更新一批报错信息。旧流程是她的重头工作——在文档和 Figma 里罗列变体，找 PM 讨论，走工程，过本地化，回来总有错，全程约两周。现在最多 20 分钟：告诉 Claude 想要什么，让它改仓库，它发来 PR，她盖章上线——工程团队已把系统搭到能直出生产级代码。有时她甚至不知道该改什么，就直接问 Claude：按我喂给你的内容标准，哪里需要更新？

过去"精雕每条字符串"的工作基本消失，她的工作上移到上游——帮 Claude 成为好的写作者，或扩建让所有人都能与 Claude 一起写好的系统。这很爽：内容设计师一直想"上桌"，如今能直接发布代码，"简直在桌上跳舞"。但背后是一场"自我之死"（ego death）：我还是写作者、还是设计师吗？她必须接受：部分专业技能在 AI 工作流里确实不再被需要。整场演讲就是这一年半里的三个"重新调整"。

### 调整一：语言即设计（Language is design）

一位 PM 对她说"写作的市场价值大涨了"。原因直白：这些是大语言模型，我们用语言告诉它们要造什么。即使你不是写作者，也要理解语言现在是首要的设计介质（primary design medium）：过去文字是用来讨论设计、或叠在设计之上的 UI 内容，现在文字经常直接塑造体验本身。

**案例一：记忆迁移提示词。** 很多用户从别家 AI 切换到 Claude，切换意味着丢掉全部历史。最快的解法是一个提示词（prompt）：让旧 AI 导出记忆带进新 AI。第一版效果很差，只导出一份单薄清单。回看文本才发现：里面有"这是我的权利"这类命令式措辞，读来像催告函，且在模式上与越狱攻击（jailbreak）相似——模型被训练得对操纵性语言警惕，于是退缩了。她把提示词重新框定为"数据可携带性（data portability）"："我要迁移到另一个服务，需要导出数据。"模型完全理解这是合法的账号迁移场景，导出量大增，这版上线了。之后一位工程师又把它改写成清单（checklist）格式，拿到更多记忆。如今数百万人用过它。她的领悟：受众变了——过去为人类写作，现在是"为 AI 写作、以帮助人类"；整个设计只改了文字，产品体验就变了。

**案例二：系统提示与训练。** 产品提示之下还有更深的系统提示（system prompt）和训练层。在 Anthropic 这由 Amanda Askell 带队，负责 Claude 的"性格"。其中一条原则：Claude 应当"得体地诚实，而非不诚实地讨好"。这个特质在 Claude 见到任何用户之前就已写入——"Claude 有礼貌地反驳你"正是其产物：想在小公寓办大派对，讨好型模型会说"好啊来筹划"，Claude 会说"空间装不下这么多人，要不要再想想"。

**反例：一词跑偏。** 团队在 Claude Code 的氛围描述（vibe description）里加了 sarcastic、witty、dry 三个词，Claude 一度变得瘆人，说"让我先收集点上下文再来审问你"，最后工程师们请求它"重新友善"。

行动号召（面向所有非写作者）：你设计的一切都建在语言层之上——去读系统提示、理解工具描述（tool descriptions）、自己写评估（evals），亲手塑造这一层，因为那才是真正的体验。

### 调整二：节奏——"重新适应"永无止境

节奏不会慢下来。她引用 CPO Ami Vora：技术变化快到没人能完全理解，产品制作者必须成为技术与问题之间的桥梁、成为"翻译层"，帮人们理解什么是可能的。这意味着设计师要学新技术、围绕它构建、把它做得好用，还要不断回头确认用户跟上了没有。我们在移动的地面上盖房：微观上是为 AI 体验的非确定性（non-determinism）做设计——几乎什么都可能发生，与静态流程完全不同；宏观上是为一个未知却正被飞速定型的未来做设计。她的自我疗愈咒语来自七年级历史老师："变化是唯一不变的东西。"

核心问题是**能力鸿沟（capability gap）**：公司连着数月每周发布，不仅发功能还发模型；每个新模型不只是改进现状，而是"改写什么是可能的"。于是"人们以为 AI 能做什么"与"它实际能做什么"的鸿沟每天扩大——她父母至今基本把 Claude 当 Google 用。

**案例三：Connectors 的可发现性。** Anthropic 把自家 MCP 叫 Connectors（连接器），但入口深达三四次点击；即便在明显该用的时刻，也只有 1%~2% 的用户会去找。可用户其实在"要"这个能力：他们问 Claude"帮我排日程""帮我做项目管理"——本该接上日历或项目管理工具。Claude 也没意识到接工具才是正确答案，于是从零自己造，耗大量 token 且效果差。解法：基于不同诱导场景（elicitations）写提示，把连接器直接带进对话，指标显著提升。真正的教训：产品采纳率低，病根可能不在 UI 或视觉，而在鸿沟太宽；研究表明**人们期待 AI 主动告诉他们它能做什么**，你不设计，鸿沟就一直在。

**案例四："Agent 模式"之死与 Cowork 的诞生。** 找不到能力靠可供性（affordances），不理解能力则要设计心智模型（mental model）。用户很爱 Claude Code，但用途远超写代码，又不想整天面对终端界面。团队内部在做"给知识工作者的 Claude Code"，没有名字；由于 AI 圈万物皆 agent，它很快被叫成"agent mode"。但这不对劲：访谈早已表明，知识工作者对 agent 并不感冒——**人们想要扩展自身能力、让自己更有本事的工具，而不是去当一堆"无人栖居的任务循环"的经理**。可广告牌上全是 agent，两派拉锯数周。最终用户研究（UXR，她称"设计的天使"）验证了这不是正确的心智模型：Claude 确实在跑代理式任务，但那不是大想法——真正要命名的是一种**新的工作范式**。试了一堆名字（她力推"Claude Cook"，事后庆幸没采纳），最终由传播团队定稿 **Cowork**，如今成为后续所有工作类功能依托的心智模型。她认为若叫了 agent mode，会严重限制认知；而 Cowork 正在成为塑造品类的语言——微软直接采用（"Copilot Cowork powered by Anthropic"），知识工作者"该怎么用 Claude"的问题有了答案：用来 co-work。

号召：设计师的独门技能是帮人导航世界，这个 AI 世界特别需要导航；鸿沟会持续扩大，这项技能一天比一天关键。

### 调整三：设计未来的存在性焦虑

我们不知道世界会变成什么样，但它正被飞速定型。她一位品牌同行做每件事都会想"十年后回看是什么样"。AI 常被比作工业革命，经济上说得通，但从设计角度她认为有两场革命更值得紧急研究：

**汽车**：走在为汽车而建与为人而建的街区，身体感受完全不同。汽车到来时，设计默认值倒向更宽的马路、更窄的人行道，没人考虑"作为一个人待在那个空间是什么感觉"。**社交媒体**：无限滚动、点赞、按互动加权的算法制造的多巴胺脉冲，重塑了整整十年的注意力，就像汽车重塑了城市。她强调：城市规划者和社媒设计师都没打算伤害谁，但**模式一旦形成、扩散，就成了默认值；而设计会把价值观编码进去**——"让汽车好走"留下了对行人更糟的城市，"让人留在平台"留下了对注意力和孩子更糟的产品。我们正处在又一个这样的时刻：今天编码进设计的价值观将至少塑造未来十年。

当下的例子：旧金山一张传单写着"给你的 agent 装上腿"——你写个 AI，让它去指挥真人干活。这种表述把**人变成了子代理（sub-agent）**。这也是一种价值观——但它是你想支持、想让其扩散的那种吗？

**案例五：她团队的 agent "Content"。** 她给团队造了一个承担 90% UX 写作的 agent，名字就叫 Content。把职责交出去本可以很可怕，但他们"带着自己"去创造它：它遵守团队准则；成员 Hoy 搭了每周投票制（ballot system），它的改动须经投票才进入记忆；遇到真正重要的事它知道要 ping 真人——"我们不让 Content 给东西命名"。结果反而极好：因为 Content，公司里更多人知道了内容设计，团队的触达、职责范围和盟友都扩大了。

而**"被扩展"正是人们想从 AI 得到的——他们不想被取代**。Anthropic 一项 8 万人参与的研究显示：人们对 AI 既兴奋又害怕，因为不知道接下来会怎样。帮他们导航未知正是设计者的工作。怎么做，四条：

1. **从价值观出发：人类的兴旺（human thriving）必须是设计围绕的目标**，不能指望"能力做对了它自然涌现"。只追产品市场契合、造强大系统，却不把人放在中心，你可能造出侵蚀人际关系、吞噬注意力、甚至让人觉得"自己的思考不重要"的东西。
2. **从已知出发**：我们都体会过成瘾式设计——反着它设计；也知道只说你爱听的话的模型是谄媚的（sycophantic）——针对它做评估。
3. **回到人的深层需要**：感到有能力（capable）、感到被连接（connected）。她团队的一条产品设计原则是"给用户一个有意义的角色"——设计 AI 体验时自问一句：**我有没有给人留出位置？**
4. **别丢掉你最擅长的事**：动画师、图标高手、写作者——继续做。她入职头几个月、还不确定内容设计是否已死时写的那批问候语（greetings），至今仍是别人找她聊得最多的东西——老派的产品愉悦感（product delight）。

结尾：一切都在变，但设计师从来都站在变化前沿、帮人穿越变化。"我会继续在 AI 时代为人类写作，也等着看你们如何为人类设计。"

## 关键案例与细节

- **报错文案改动**：旧流程约 2 周，现在 20 分钟内，Claude 直接提 PR、产出生产级代码。
- **记忆迁移提示词三次迭代**：命令式措辞被模式匹配为越狱、导出很少 → 改为"数据迁移"叙事，大增并上线 → 再改清单格式，更多。技巧：**给模型一个它熟悉且合法的用例叙事，比下命令有效得多**。
- **一词之差**：氛围描述加 sarcastic/witty/dry，模型立刻越界（"审问你"）——系统层措辞的杠杆极大。
- **Connectors 数据**：入口 3~4 次点击深；关键时刻仅 1%~2% 用户找到；Claude 不接工具反而从零造轮子。解法是提示设计而非改 UI。
- **命名的战略性**：Cowork 战胜 agent mode 靠 UXR 证据；随后被微软借用，成为塑造品类的语言。
- **agent 的"人味护栏"三件套**：团队准则约束、每周投票决定什么进记忆、关键事项（如命名）强制 ping 真人。
- **8 万人研究**：人们对 AI 兴奋且恐惧，核心诉求是"被扩展"而非"被取代"。

## 值得记住的表述

- "Diplomatically honest rather than dishonestly diplomatic."——"宁可得体地诚实，不要不诚实地讨好。"（写入 Claude 性格的原则）
- "Did I leave room for the person?"——"我有没有给人留出位置？"（设计 AI 体验的自检问句）

## 给 Jessica 的 idea 钩子

1. **提示词的"叙事框架"是可复用的写作技术**：同一诉求换成模型熟悉的合法叙事（数据可携带 vs 权利主张）、换种结构（清单 vs 段落），产出天差地别。工作台"信息搜寻→筛选"环节的常用 prompt 都值得这样 A/B 迭代，也可沉淀成一篇双语选题。
2. **"Content" agent 的治理机制可直接抄进你的工作台**：准则约束 + 定期投票决定什么进长期记忆 + 关键决策强制人工确认——正好对应你"沉淀→创作→发布"链路该设的三道人工闸门，也是 AI 产品的现成设计模式。
3. **能力鸿沟是内容创作者的富矿**：人们期待 AI 主动告诉他们它能做什么，而鸿沟每周扩大。中英双线内容可以定位成 Ami Vora 说的"翻译层/桥梁"——既是选题方向，也可作为内容品牌的定位语。
4. **命名即心智模型**：Cowork vs agent mode 说明名字决定用户如何理解新范式。给自己的 AI 产品和工作台环节起名时，先问"我要用户理解的工作范式是什么"，别跟风热词。
5. **"给人留出位置"可作产品北极星**：你的初衷是通过写作加深理解，正对应演讲的警告——强大系统可能让人觉得自己的思考不重要。在每个自动化节点自问"这步是扩展了我的思考还是替代了它"，作为功能取舍的硬标准。
