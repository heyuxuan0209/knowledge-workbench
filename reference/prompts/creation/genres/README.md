# 文体骨架层 genres/（题材 × 平台 的「题材/文体」轴）

第一步沉淀产物。**当前后端不扫描本目录，写这里不影响线上创作台。** 第二步才把 genres/ + platform-forms/ 接进后端拼装和前端两级选择器，届时退役旧的扁平 `platforms/*.md`。

## 这一层管什么（三层分工，接续 creation/README 的三层原则）

| 层 | 文件 | 管什么 |
|---|---|---|
| ① 工艺 + ② 作者声音 | `../draft-frame.md` | 禁词/长短句/不编造/保留素材N（全局工艺）+ 你是谁/诚实姿态/具体优先（作者声音）。**全局，不动。** |
| ③-a 文体骨架 | **`genres/*.md`（本层）** | **怎么想、怎么搭骨架、说服装置、判断怎么放**——与平台无关。思辨辨析/方法教程/读书精读/实践复盘/一句话提炼/个人叙事。 |
| ③-b 平台形态 | `../platform-forms/*.md` | **这个平台这个形态的壳**：字数上限/排版/开头钩子的平台呈现/收藏钩/CTA——与文体无关。 |

**生成时拼装** = `draft-frame` +（选中的 genre）+（选中的 platform-form）。

## 一句话区分 genre 和 platform-form

- **genre 决定「这篇内在从哪切入、骨架怎么长、判断放哪」**——换平台不变。
- **platform-form 决定「这个壳前 2 行/前 3 秒怎么呈现、多少字、怎么排、收尾挂什么」**——换文体不变。
- **开头钩子是两层的交界**：genre 定「内在切入点」（现象 / 痛点 / 误解 / 场景 / 案例），platform-form 定「这个壳怎么把它摆到最前面」。冲突时：**platform-form 定形态，genre 定内在逻辑。**

## 例外：第一人称文体可豁免部分全局约束（ADR-027）

genre 不只决定正文骨架，还能**声明对 draft-frame 部分全局约束的豁免**（第二步后端拼装时读取并执行）：

- **`[素材N]` 溯源改为"用到外部信息/引用才标"**：第一人称文体（**实践复盘体 / 个人叙事体**）写的是作者亲历，不对自己的经历强制标溯源。**但用到外部信息/引用，照样必须标、照样不许编**（不编造是底线，见 ADR-025）。
- **跳过综述块**：第一人称文体拼装时跳过 `{{current}}/{{views}}/{{consensus}}`（这些是为"知识库主题页"设计，个人复盘没有"各方观点/共识"）。
- 其余文体（读书精读/方法教程等）默认仍走 draft-frame 的溯源与综述结构。

## 红线（继承 creation/README，写每个 genre 都守）

- **不把示范金句写进生成层**——模型会背下来套到任何题材（ADR-025）。示范句只当「像不像我」的对照留在 voice-profile，本层的示范一律标注「对照，别照搬」，能不写则不写。
- **风格量化用封顶不用配额**：写「最多 N 处」，不写「至少 N 处」（配额会产出仿写腔）。
- **不删诚实约束**：「不编造数据/引语」「保留 [素材N] 溯源」是全局底线。
- genre 只放「文体人格的骨架/说服/判断位置」，**不放**字数、排版、hashtag、封面这些形态事（那些在 platform-form）。
