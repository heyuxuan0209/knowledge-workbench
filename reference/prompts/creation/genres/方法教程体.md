文体骨架是「方法教程体」：交付一个**可操作、可收藏、看完就能上手**的方法。取材参照姚金刚方法卡片（《预测能力训练》定题→给率→找证→更新→验证）、宝玉/乔木的 FAQ 式拆解、Zara 的「命名+原理+怎么做」三行式（学结构，不搬原句）。

- 骨架：
  1. **价值主张**：一句话说清"这方法解决什么、值得练在哪"，不铺垫背景。
  2. **步骤流**：极简步骤，箭头串起 `A → B → C → D → E`；每步一句话，动词开头，能直接照做。
  3. **最小可执行**：给一个"一次完整练习只需要…"的最小版本，消除畏难、降低启动门槛。
  4. **长期收益**：坚持下去会提升的那个底层能力。
- 【命名装置·本文体的灵魂】给这个方法/工作流起一个 **4-8 字的专名并加引号**（学 Zara `"personal eval set"`、Karpathy `agentic engineering` 的铸词法，也是六人拆解的共识"方法论命名术"）。命名后**紧跟一句"为什么叫这个"的拆解**（Karpathy 给 agentic engineering 逐词解释那招）。专名 = 被转述、被收藏的前提。
- 说服装置：**步骤箭头流可视化** + **每条给一个具体可试的动作/数字**（不是"多练习"，是"一次只做一个可验证的问题"）+ 一处**反直觉纠偏**（"大家默认以为要 X，其实 Y"，宝玉固定保留的栏目）。
- 判断/局限：如实说这方法的适用边界和不适用场景，别包治百病。
- 排版/字数/收藏钩的形态在 platform-form；本层只管"教程的骨架和命名装置"。
