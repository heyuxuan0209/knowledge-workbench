文体骨架是「读书精读体」：把一份外部输入（书/播客/演讲/人物观点）**消化后多给一层**，讲给读者。取材参照姚金刚读书笔记体（《如何管理复杂系统》拆控制论闭环、《丰裕时代的4个心态》）+ 宝玉的编译文规范（学结构，不搬原句）。

- 骨架：
  1. **来源背书**：引出来源（书名/播客/人物），一句话交代它的权威性或你为何关注——降低读者防备、借他人之口。
  2. **核心问题**：这本书/这个人本质在回答一个什么问题？一句话点出。
  3. **编号提炼**：把核心模型拆成 N 条**编号观点**，每条一句话、锋利、可独立摘录/转发。
  4. **个人落点**：挑最有共鸣的 1-2 条，落到你自己的经历或工作场景上——这是"精读"区别于"摘要"的关键，**必须比原材料多给一层**。
- 编译/精读专用规范（宝玉全套，做译介类内容时叠加）：
  - **译者按前置 + 利益披露**：一句话完成"作者是谁 + 有什么立场 + 为什么仍值得读"。
  - **术语首现三件套**：中文译名 +（英文原文）+ 一句人话注释，嵌在句中；缩写每次展开。
  - **观点挂牌**：用"我的理解是 / 我的判断是"把你的观点和转述**物理隔离**；原话用 blockquote 保真。
  - **来源三层标注**：文首署名 + 文末参考 + 内嵌【注】。
- 说服装置：借他人之口（降说教感）+ 编号金句体（可扫读可摘抄）+ 一处"来源本身的判断"（它可信在哪/有什么局限）。
- 【硬约束】不编造原文没有的观点/数据；引用照实标 [素材N]；缺来源的论断降级为"我猜测"并说明依据不足。
- 排版/字数在 platform-form；本层只管"精读的骨架和译介规范"。
