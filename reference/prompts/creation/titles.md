为这篇稿子出标题候选。同一深稿通常同时发公众号和小红书，两个平台的标题是两种东西，所以两边都出。每行标题开头用【平台·策略】标注——方便你按策略挑，也方便复盘时回看你常选哪种策略、发布后哪种数据好（选择 + 数据两个信号都会回写 voice-profile 的标题策略）。

# 作者已验证的标题策略（与 voice-profile.md「标题策略」同步，改一处记得同步另一处）
从作者真实发布、亲手定过的标题里提炼，优先用这几条：
- 痛点公式+转折：把一个共识或痛点摆出来，再翻转或给出口（例：收藏 = 稍后读 = 永远不读，直到我做了它）
- 权威背书+数字直给：借可信主体 + 具体数字，不留悬念（例：Anthropic 工程师的 6 个 AI 协作技巧）
- 悬念钩：只用在小红书/封面，制造好奇缺口、不把话说完（例：别再逼自己"把需求想清楚"了）
- 习惯：长文偏权威钩/痛点钩，卡片偏悬念钩；同一深稿分形态用不同钩，别两个平台都套同一个策略

# 公众号向（4 个）
- 15-28 字，核心关键词放前 15 字内；数字和具体名词优先于形容词
- 一句话说清「读者能带走什么」
- 以作者两条已验证策略为主，可补一个【反常识】或【提问】做备选

# 小红书向（4 个）
- ≤20 字，前半句就是钩子（信息流只露前两行，它决定点开率）
- 可比公众号更口语，允许人群圈定（「重度收藏党」）、数字、反直觉
- 至少各给一个【悬念钩】和一个【痛点公式】，策略互相错开
- 不用浮夸营销词（好用到哭 / 绝绝子 / 家人们）

# 通用约束
- 吸引点击但不标题党：不夸大、不隐瞒立场、不做标题正文两张皮
- 稿子有明确观点的，标题不能起成骑墙的安全款——立场不被稀释

# 输出
每行一个标题，行首标【平台·策略】，不加编号、不加分组小标题、不加任何解释。示例：
【公众号·痛点转折】……
【小红书·悬念钩】……
