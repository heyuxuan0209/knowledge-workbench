平台形态壳：**X 单条短推**，中英各出一版（不是互译，各按各自语感重写）。取材 docs/teardowns/x-chinese.md、x-english-vc-writers.md（Dan/Zara）、x-english-karpathy.md。冷启动**优先收藏型实操推，不做情绪爆款**（bookmark 抗算法、长尾流量）。

> **【文风·实测校准 2026-07-19】X 短推走「克制工程师」基调：具体、直给、落到真事。**
> **不要抽象金句 / 思辨腔 / 卖弄才情**——那种在公众号长文里是"有想法"，压到短推就成了装（实测："判断权藏在可见里"这类点睛金句被判为卖弄）。
> 开头给**具体锚点**（上周 / 两次 / 一件真事），收尾给**一条平实的规矩**，不要点睛式金句收尾。（长文可用 A 克制 + B 思辨；短推只用 A，不用 B。）

- **单条自足**：一条说清一件事，不依赖上下文；≤280 字符（英文），中文约 ≤140 字。
- **头号骨架·命名三行式**（Zara + Karpathy 铸词法，你知识管理赛道性价比最高的一招）：
  `第一行：给工作流/现象起个加引号的 2 词名` → `第二行：一句原理` → `第三行：一句怎么做`。命名后可紧跟一句"为什么叫这个"。
- **其他可选句式**（选一种，别叠）：冒号铺垫+空行+落点 / 对话体 `People:／Me:` / 极短格言体（一句判断）/ 反直觉断言+具体数字（学 Dan `4 -> 30`）/ "被严重低估 · 最大风险其实是 X 不是 Y"判断框架。
- **中文版**：一句一段、空行分隔；**零 hashtag、零 emoji**；口语语气词可留（嘿嘿/哈哈）；去 AI 味靠"交代过程而非只下结论"+ 具体版本号/数字。
- **英文版**：句首常小写、口语缩写（tbh/ofc/imo）、emoji 克制到零；引号专用于自造概念；`slop`-free（少装饰多留白）。
- **引流（可选）**：主推讲价值，把链接放在**自回复**里，不塞进主推（学 Dan）。
- **避开**：引战大厂、求转发、晒战报、蹭人脉——依赖体量，小号照抄翻车。
- 骨架由所选 genre 决定（一句话提炼体/方法教程体最配）；本文件只定 X 短推这个壳。
