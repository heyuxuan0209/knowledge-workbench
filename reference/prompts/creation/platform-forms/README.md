# 平台形态层 platform-forms/（题材 × 平台 的「平台发布形态」轴）

第一步沉淀产物。**当前后端不扫描本目录**（线上仍走旧的 `../platforms/*.md`），写这里不影响创作台。第二步把 genres/ + platform-forms/ 接进后端拼装 + 前端两级选择器，退役旧 platforms/。

## 这一层管什么

一个文件 = **一个「平台 × 形态」发布单元**。只管**壳**：字数上限 / 排版 / 开头钩子在这个壳里怎么呈现 / 收藏钩 / CTA / 该平台传播工艺。**不管**骨架和说服（那在 genres/）。

生成时拼装 = `../draft-frame.md` +（选中 genre）+（选中 platform-form）。

## 形态（form）是跨平台复用的概念

同一「形态」的通用体例在多个平台复用，各文件里已按平台调性各自写清，但共享同一套 DNA：

| 形态 | 通用 DNA | 出现在 |
|---|---|---|
| 长文 | 小标题/排版/收藏钩/来源标注 | gzh-long, xhs-long, reddit |
| 卡片图文 | 一屏一点/封面钩/字数极限 | xhs-card, douyin-card |
| 口播视频 | 前段留人/口语/过程叙事/节奏标注 | douyin-koubo, bilibili |
| 短内容 | 单条自足/钩子前置/克制排版 | x-short, jike |
| 长推/thread | 首条钩子/编号或分章/引流结构 | x-thread |

## 各平台字数/形态硬上限（红线，别越）

| 文件 | 平台·形态 | 硬约束 |
|---|---|---|
| gzh-long | 公众号·长文 | 1500-3500 字，Markdown |
| xhs-long | 小红书·长文 | 正文 ≤1000 字符（小红书硬上限） |
| xhs-card | 小红书·卡片图文 | 含封面 ≤9 张；封面大字 ≤12 字；卡正文 ≤60 字 |
| douyin-koubo | 抖音·口播（长讲解） | 无硬字数，按江江"长讲解"体，口语 |
| douyin-card | 抖音·卡片图文 | 图文卡，一屏一点 |
| x-short | X·短推（中/英） | 单条 ≤280 字符（英文），中文约 ≤140 字 |
| x-thread | X·长推（中/英） | 中文优先"单条长推内编号"，英文可 thread |
| bilibili | B站·视频 | 口播/录屏脚本，可分 P，更长 |
| jike | 即刻·短思考 | 短，圈子氛围 |
| reddit | reddit·帖（中/英） | 社区体，标题不党，正文长且个人化 |

## 最高原则：价值优先于爆款（过犹不及）

**所有平台形态里的"钩子/点开率/收藏钩/封面"都是手段，不是目的。真实价值是目的。** 依据：X（马斯克开源的算法）明确对"制造爆款标题但没真实创造用户需求"的内容降权，把权重给真正带来价值的内容；坚持稳定输出价值 > 追求单篇爆款（也是江江内容观、姚金刚反流量的共识）。

落到每个 platform-form：
- 钩子/封面只需**如实点出这篇的真实价值和适用的人**，不做"太炸裂了/再不学就晚了"式的夸张与制造焦虑。
- **宁可钩子平实，也不要标题党**——标题与正文价值不匹配，是负资产。
- 收藏钩来自"内容真的可复用"，不是套路装置；讲不出真实增量的卡/段直接删。
- 冷启动补 CTA 时用**利他理由**（"这对你可能有用"），不用求赞/求转发/诱导互动。

## 红线（继承 creation/README）

不编造数据/引语、保留 [素材N]；不把示范金句写进模板；风格量化用封顶不用配额；焦虑贩卖/诱导互动/搬运洗稿/假装体量一律禁（见 platform-playbooks 负面清单）。
