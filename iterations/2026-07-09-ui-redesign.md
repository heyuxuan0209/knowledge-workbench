## 重大决策

### 决策 1: 主题色方案

| 方案 | 优点 | 缺点 | 结果 |
|------|------|------|------|
| 深色（#1a1a1a） | 酷炫，程序员喜欢 | 阅读疲劳，不够正式 | ❌ 放弃 |
| 浅色米白（#fafaf9） | 专业，舒适，主流 | 略显平淡 | ✅ 采纳 |
| 纯白（#ffffff） | 最简洁 | 对比度过高，刺眼 | ❌ 放弃 |

**最终选择**: 浅色米白 `#fafaf9` + stone 色系  
**原因**: 
- 参考 Claude 官网和 AI HOT 的设计
- 长时间阅读更舒适
- 专业感更强

### 决策 2: 图标库选择

| 方案 | 优点 | 缺点 | 结果 |
|------|------|------|------|
| Emoji | 零成本，色彩丰富 | 不专业，风格不统一 | ❌ 放弃 |
| Heroicons | Tailwind 官方推荐，简洁 | 图标数量较少 | ✅ 采纳 |
| Lucide | 图标数量多，设计精美 | 需要额外安装 | 备选 |
| FontAwesome | 图标最多 | 体积大，风格偏重 | ❌ 放弃 |

**最终选择**: Heroicons（内联 SVG）  
**原因**:
- Tailwind CSS 官方推荐
- 无需额外依赖
- 风格简洁统一

### 决策 3: 布局架构

**选择**: 三栏固定式（类似 Claude / Cursor）  
**原因**:
- 用户习惯了对话式工作台
- 信息密度高
- 左侧导航 + 中间内容 + 右侧面板的模式已被验证

## 踩坑记录

### 坑 1: Tailwind CSS 样式不生效

**问题**: 
修改组件后保存，页面样式突然全部丢失，只剩下纯 HTML

**原因**: 
`tailwind.config.js` 的 `content` 路径配置不包含新创建的 `components/workspace/` 目录

**解决**:
```javascript
// tailwind.config.js
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",  // 确保包含所有子目录
  ],
  // ...
}
```

**预防**: 
每次创建新目录后，检查 Tailwind 配置的 `content` 数组

---

### 坑 2: AI HOT API 数据不完整

**问题**: 
`/api/public/items` 端点只返回基础字段，没有：
- 完整正文（富文本 HTML/Markdown）
- 推荐理由（精选原因）
- 真实标签（文章提取的标签）
- 多信源聚合

**影响**: 
详情页无法显示完整内容，推荐理由和标签是 mock 数据

**临时方案**: 
1. 推荐理由：硬编码一段通用文案
2. 标签：手动写几个示例标签
3. 正文：显示摘要 + 提示"需要接入详情 API"

**长期方案**: 
研究 AI HOT 是否有详情端点，或者爬取网页内容

---

### 坑 3: React 组件导入路径错误

**问题**: 
```javascript
import ArticleDetail from './ArticleDetail'
// Error: Cannot find module './ArticleDetail'
```

**原因**: 
文件实际在 `components/workspace/ArticleDetail.jsx`，但从 `pages/` 导入时路径写错了

**解决**: 
使用相对路径：
```javascript
import ArticleDetail from '../components/workspace/ArticleDetail'
```

**预防**: 
使用 VS Code 的自动导入功能，或者配置路径别名

---

### 坑 4: 中英文切换逻辑混乱

**问题**: 
最初理解错了需求，以为是切换界面语言（把"返回列表"改成"Back"）

**正确理解**: 
切换的是**文章正文内容**的语言：
- 中文译文：显示 AI HOT 提供的中文翻译
- English 原文：显示英文原文

**解决**: 
只在有 `title_en` 且不同于 `title` 的文章上显示切换按钮
