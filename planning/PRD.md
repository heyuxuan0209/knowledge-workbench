## 一、产品定位

### 1.1 核心定位

**不是**: AI 资讯聚合器（AI HOT 已经做了）  
**而是**: 个人化的 AI 信息工作台 + 知识炼金术

### 1.2 与 AI HOT 的差异

| 维度 | AI HOT | AI Insight Hub |
|-----|--------|----------------|
| 定位 | 大众信息广场 | 个人工作台 |
| 内容量 | 100+ 条/天 | 15-20 条精选/天 |
| 筛选 | 通用算法 | 学习个人偏好 |
| 组织方式 | 分类标签 | **主题追踪**（周/月维度）|
| 深度功能 | 无 | **研究工作区**（多篇对比分析）|
| 项目关联 | 无 | **GitHub 项目匹配** |
| 知识沉淀 | 无 | 导出到 Obsidian |
| 创作支持 | 无 | 一键发送到 ChatGPT/Claude |

### 1.3 核心价值链

```
AI HOT 信息源 (100+条/天)
    ↓ 个性化筛选
精选推荐 (15-20条)
    ↓ 主题自动聚类
主题追踪 (周/月维度)
    ↓ 深度研究
多篇对比分析 + GitHub项目匹配
    ↓ 创作辅助
发送到 ChatGPT/Claude 或 导出 Obsidian
    ↓ 知识沉淀
内容创作 + 产品构思
```

## 二、核心使用场景

### 场景1: 主题追踪 + 深度研究 + 二次创作

**用户故事**:
> 作为一名 AI 产品人，我想从 AI HOT 每天的推文中找到我感兴趣的话题，把同一话题的推文、博文等放在一起，以周/月为单位追踪某个 topic，挑选出一些进行深度研究，激发我的二次创作。我可能会在产品中直接使用 LLM 创作，也可能跳转到 ChatGPT、Claude 链接中让他们帮我分析这些文章，帮助我创作。

**关键功能**:
1. 从 100+ 条内容中快速筛选出感兴趣的
2. 自动识别同一主题的内容并归类
3. 追踪主题的演进（本周/本月新增多少条）
4. 选中多篇内容进行对比分析
5. 一键发送到 ChatGPT/Claude
6. 导出到 Obsidian 作为创作素材

### 场景2: GitHub 项目发现 + 产品启发

**用户故事**:
> 通过推送的 GitHub 高星和热门项目，追踪的 X 平台 AI builders，对我的产品产生一些启发。我不想从0造轮子，而是想知道市面上有哪些开源项目可以复用或者二次魔改，实现我的目的。

**关键功能**:
1. GitHub 高星项目自动推送
2. X 平台 AI builders 动态追踪
3. 基于我的需求/想法，自动匹配相关的开源项目
4. 评估项目可复用性（匹配度、技术栈、改造成本）
5. 关联相关的文章和讨论

### 4.2 主题识别算法

**混合方案**（关键词 + LLM 验证）:

```javascript
// 伪代码
async function identifyTopics(items) {
  // Step 1: 本地关键词提取（快速，成本为0）
  const keywords = items.map(item => 
    extractKeywords(item.title + item.summary, topN: 10)
  );
  
  // Step 2: 余弦相似度计算（本地）
  const similarities = calculateCosineSimilarity(keywords);
  
  // Step 3: 初步聚类（本地，基于相似度阈值 0.75）
  const clusters = clusterBySimilarity(similarities);
  
  // Step 4: LLM 验证和命名（仅对候选主题，成本可控）
  const topics = await Promise.all(
    clusters
      .filter(cluster => cluster.length >= 2) // 至少2篇
      .map(async cluster => {
        const result = await llm.analyze({
          model: "deepseek-chat", // 使用低成本模型
          prompt: `
            以下 ${cluster.length} 篇文章是否属于同一主题？
            
            ${cluster.map((item, i) => 
              `${i+1}. ${item.title}\n   摘要: ${item.summary}`
            ).join('\n\n')}
            
            返回 JSON:
            {
              "is_same_topic": true/false,
              "topic_name": "主题名称（如果是同一主题）",
              "confidence": 0-1,
              "keywords": ["关键词1", "关键词2", "关键词3"]
            }
          `
        });
        
        return {
          ...result,
          items: cluster
        };
      })
  );
  
  // 返回置信度 >0.8 的主题
  return topics.filter(t => t.is_same_topic && t.confidence > 0.8);
}
```

**成本估算**:
- 每天 100 条内容
- 可能识别出 10-15 个候选主题
- 每个主题验证成本: ~$0.001
- 每日成本: ~$0.01-0.02

### 4.3 偏好学习

**数据收集**:
- 用户点击"感兴趣" / "不感兴趣"
- 用户创建主题追踪
- 用户移入研究工作区
- 用户保存到 Obsidian
- 用户跳过/忽略

**学习策略**:

```javascript
// 权重更新
const updatePreferences = (action, item) => {
  switch(action) {
    case 'interested':
      increaseWeight(item.keywords, +0.15);
      increaseWeight(item.category, +0.10);
      break;
    case 'not_interested':
      decreaseWeight(item.keywords, -0.10);
      break;
    case 'saved':
      increaseWeight(item.keywords, +0.20);
      increaseWeight(item.category, +0.15);
      break;
    case 'skipped':
      decreaseWeight(item.keywords, -0.05);
      break;
  }
};

// 推荐评分
const calculateRecommendScore = (item, userPreferences) => {
  let score = 0;
  
  // 1. 关键词匹配 (40%)
  item.keywords.forEach(keyword => {
    score += (userPreferences.keywords[keyword] || 0) * 0.4;
  });
  
  // 2. 分类匹配 (30%)
  score += (userPreferences.categories[item.category] || 0) * 0.3;
  
  // 3. AI HOT 原始评分 (20%)
  score += (item.score / 100) * 0.2;
  
  // 4. 来源权重 (10%)
  score += (userPreferences.sources[item.source] || 0) * 0.1;
  
  return Math.min(100, Math.max(0, score * 100));
};
```

## 八、设计决策记录

### 决策 #001: 为什么三栏布局？

**背景**: 用户有两个核心场景（主题追踪 + 项目匹配）

**决策**: 采用三栏布局（左-主题，中-内容，右-工作区）

**理由**:
- 左侧主题列表 → 满足"周/月追踪"需求
- 中间内容池 → 满足"同话题聚合"需求
- 右侧工作区 → 满足"深度研究"需求
- 信息密度高，适合有足够时间投入的用户

### 决策 #002: 为什么不做极简 MVP？

**背景**: 用户明确表示"不要再做极简的 MVP 了"

**决策**: 直接做完整的主题工作流系统

**理由**:
- 用户有足够时间投入
- 用户需要"能用起来"的完整产品
- 包含 Web 前端 + 后端 + 反馈学习

### 决策 #003: 主题识别方案

**背景**: 纯 LLM 成本高，纯关键词不准确

**决策**: 混合方案（关键词筛选 + LLM 验证）

**理由**:
- 关键词快速筛选候选主题（成本为0）
- LLM 只验证候选主题（成本可控）
- 用户可以手动调整（提高准确性）

### 决策 #004: 复用 AI HOT 数据

**背景**: 不要重复造轮子

**决策**: 直接使用 AI HOT API

**理由**:
- 已有中文摘要（避免重复翻译成本）
- 已有分类和评分
- 已包含多数信息源
- 可以专注于"筛选+主题追踪+深度研究"
