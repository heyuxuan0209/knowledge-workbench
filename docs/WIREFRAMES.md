## 核心设计原则（贯穿所有页面）

1. **一切向"产出带观点的内容"收敛**：每个界面都要能回答"这如何帮用户走向创作"。
2. **Feed 默认是全量流**，不是"关注列表"。已登记源是排序加权和"已关注"标记，不是过滤条件。
3. **Source 是优质源登记处**（ADR-007）：登记效果只有"进 Feed + 高权重"两个，
   track_mode 四档由平台类型自动判定，界面不许诺做不到的抓取能力。
4. **日报/周报/月报是节奏化简报**（ADR-008），不实时刷新；焦点和选题只基于用户自己的信息流。
5. **Topic 是活页不是文件夹**（ADR-009）：正文由 AI 同化维护，演进时间线从 changelog 涌现。
6. **对话不落库，沉淀靠主动保存**（ADR-010）：素材卡片 = 结构化摘录 + 来源引用。
7. **即兴分析必须基于原文**：article/paper/repo 选中时实时抓原文，失败降级摘要且显式标注
   "无法获取原文，以下基于摘要"——不能悄悄降级（已在 ephemeral-chat.js 实现）。
8. **创作产物每段可溯源**（ADR-011）：无素材支撑的段落要显式警示。

## 0. 整体信息架构（导航地图 × 六阶段流水线）

```
                        ┌──────────────────────────────────────────┐
                        │              左侧导航（常驻）              │
                        ├──────────────────────────────────────────┤
 六阶段流水线            │  📰 Feed        ── ①采集+④洞察（日报置顶）│
                        │  💾 素材库      ── ③沉淀（素材卡片）  ✅  │
 ①采集 → ②理解 →        │  👥 信息源      ── ①采集（登记处）    ✅  │
 ③沉淀 → ④洞察 →        │  🗂️ Topics      ── ④洞察（活页）      📋M3│
 ⑤创作 → ⑥发布          │  📊 周报/月报   ── ④洞察（涌现）      📋M3│
                        │  ✍️ 创作台      ── ⑤创作+⑥发布       📋M4│
                        │  ⚙️ Settings                              │
                        └──────────────────────────────────────────┘
   右侧 AI 面板（常驻，可折叠）= ②理解：Quick Analysis / 对话 / 保存到笔记 ✅

 内容流转（飞轮）：
 Feed/丢链接 → 选中分析(对话) → 保存到笔记(素材卡片) → 归入 Topic 活页
     ↑                              ↓ 加为信息源              ↓
 登记源加权推送 ←──────────────── 信息源登记处      日报选题 → 创作台 → 导出发布
```

## 12. 批量导入信源（🔨 M2 可选）

用于：用户已在其他平台关注了一批账号，一次性导入登记处。

```
┌──────────────────────────────────────────────────────────────┐
│ 批量导入信源                                          [关闭 ×] │
│ ────────────────────────────────────────────────────────────  │
│ 从文件导入: [上传 .csv / .txt]（每行一个链接或公众号名称）    │
│ 或粘贴多行：                                                  │
│ ┌────────────────────────────────────────────────────────┐   │
│ │ https://x.com/karpathy                                  │   │
│ │ https://simonwillison.net/                              │   │
│ │ 晚点LatePost                                            │   │
│ └────────────────────────────────────────────────────────┘   │
│ [解析]                                                        │
│ ────────────────────────────────────────────────────────────  │
│ 已识别 12 个信源：                                            │
│  • 7 个 active-query（X/YouTube）  • 2 个 active-rss（博客） │
│  • 2 个 link-only（公众号）        • 1 个无法识别 ⚠ 显式列出 │
│ [确认导入全部]  [仅导入可追踪的]              [取消]          │
└──────────────────────────────────────────────────────────────┘
```

要点：解析结果按 track_mode 四档分类展示，不隐藏"仅标记跳转"的限制；
无法识别的条目显式列出，不静默丢弃。复用单条识别的 identifyInput 逐行执行。
